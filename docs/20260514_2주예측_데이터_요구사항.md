## 2. 파생 데이터 (프론트엔드 계산)

| 파생 항목 | 계산 방법 | 용도 |
|----------|----------|------|
| 터빈별 일 총 발전량 | #9 일별 24시간 합산 | 히트맵 셀 값 |

| 터빈별 14일 총 발전량 | 일 총 발전량 합산 | 랭킹 |
| O&M 종합 상태 | #4~#6 median 기반 판단 | O&M 요약 테이블 |

## 3. 데이터 제공 옵션

### 옵션 A: 최소 필수 (시간별 raw만 제공)

서버에서 제공하는 항목: **#1, #2, #3, #9, #10** (5개)

- 프론트에서 박스플롯 통계(#4~#6), 발전량 합산(#7, #8) 모두 계산
- 장점: 서버 부담 최소
- 단점: 프론트 계산 부하 (14×24×20 = 6,720개 값 처리)

### 옵션 B: 권장 (서버 계산 포함)

서버에서 제공하는 항목: **#1~#10** (10개 전부)

- 박스플롯 통계, 발전량 합산을 서버에서 미리 계산
- 장점: 프론트 부하 감소, 데이터 일관성 보장
- 단점: 서버 후처리 필요

### 권장: 옵션 B

## 4. 파일 형식 및 구조

### 4.1 디렉토리 구조

```
data/2week/
├── {발전소명}/
│   ├── YYYY-MM-DD_plant.json     ← 예측 수행일 기준
│   └── YYYY-MM-DD_turbines.json  ← 예측 수행일 기준
```

### 4.2 `{날짜}_plant.json` 구조

```json
{
  "forecast_date": "2026-05-14",
  "plant": "서남해",
  "period": { "start": "2026-05-15", "end": "2026-05-28" },
  "weather": {
    "hourly": {
      "wind_speed": [[7.2, 8.1, ...], ...],   // float[14][24]
      "temperature": [[15.3, 14.8, ...], ...], // float[14][24]
      "wave_height": [[1.2, 1.1, ...], ...]    // float[14][24]
    },
    "daily_stats": {
      "wind_speed": [
        {"min": 5.1, "q1": 6.8, "median": 8.2, "q3": 9.5, "max": 12.1},
        ...  // 14개
      ],
      "temperature": [{"min", "q1", "median", "q3", "max"}, ...],
      "wave_height": [{"min", "q1", "median", "q3", "max"}, ...]
    }
  },
  "power": {
    "daily_total": [980, 1120, ...],  // int[14] MWh
    "weekly_total": [7200, 7800]      // int[2] MWh
  }
}
```

### 4.3 `{날짜}_turbines.json` 구조

```json
{
  "forecast_date": "2026-05-14",
  "plant": "서남해",
  "period": { "start": "2026-05-15", "end": "2026-05-28" },
  "turbines": {
    "WTG01": {
      "hourly": {
        "power": [[2.1, 2.3, ...], ...],      // float[14][24] MWh
        "wind_speed": [[7.5, 8.0, ...], ...]   // float[14][24] m/s
      }
    },
    "WTG02": { ... },
    ...
    "WTG20": { ... }
  }
}
```

## 5. O&M 판단 기준 (프론트 적용)

| 변수 | O&M 가능 기준 | 초과 시 |
|------|-------------|---------|
| 풍속 | ≤ 10 m/s | 🔴 O&M 불가 |
| 파고 | ≤ 1.5 m | 🔴 O&M 불가 |
| 최고기온 | ≤ 30°C | 🟡 조건부 |
| 최저기온 | ≥ 5°C | 🟡 조건부 |

---

## 6. 데이터 로드 규칙

1. 페이지에서 선택한 날짜(예측 수행일)로 파일 fetch 시도
2. 파일 존재 → 실데이터 표시 (파란 "예측 데이터" 배지)
3. 파일 미존재 (404) → 랜덤 데이터 fallback (빨간 "SAMPLE" 배지)

## 3. O&M 결과 디스플레이 상세

> 목업 참조: https://wpf2025.github.io  
> UI 설계 상세: [WPF_UI설계_상세문서.md](WPF_UI설계_상세문서.md)

### 3.1 하루전 예측 (D+1) 화면

VPP 하루전 모델의 결과를 O&M 관점에서 재구성하여 표시합니다. 별도 모델 없이 **결과 디스플레이만 신규 개발**합니다.

#### 3.1.1 발전소 탭 — 사용자가 보는 화면

| 영역 | 표시 내용 | 비고 |
|------|----------|------|
| KPI | 내일 예상 총 발전량 (GWh) | 우측 상단 배지 |
| 복합 차트 | 시간별 발전량 (Bar, 좌축 MWh) + 풍속 (Line, 우축 m/s) | 24시간, 듀얼 Y축 |
| 날짜 선택 | `◀ YYYY-MM-DD ▶ [오늘]` | 과거/미래 날짜 조회 |

**필요 API 데이터:**

| # | 항목 | 단위 | 시간 해상도 | 비고 |
|---|------|------|-----------|------|
| 1 | 시간별 발전량 | MWh | 1시간 × 24 | 발전소 전체 합산 |
| 2 | 시간별 풍속 | m/s | 1시간 × 24 | 발전소 대표 풍속 |
| 3 | 일 총 발전량 | GWh | 일간 | KPI 표시용 (= #1 합산) |

#### 3.1.2 터빈별 탭 — 사용자가 보는 화면

| 영역 | 표시 내용 | 비고 |
|------|----------|------|
| 그리드 | WTG #1 ~ #20 개별 차트 (2열 × 10행) | 총 20개 |
| 각 차트 | 발전량 (Line, 좌축 MW) + 풍속 (Line 점선, 우축 m/s) | 듀얼 Y축, 24시간 |

**필요 API 데이터:**

| # | 항목 | 단위 | 시간 해상도 | 비고 |
|---|------|------|-----------|------|
| 1 | 터빈별 시간별 발전량 | MW | 1시간 × 24 × 20호기 | |
| 2 | 터빈별 시간별 풍속 | m/s | 1시간 × 24 × 20호기 | |

#### 3.1.3 DDI 결정/요청 사항

| # | 항목 | 설명 | 우선순위 |
|---|------|------|---------|
| D1-1 | **API 응답 형식** | 발전소 전체 + 터빈별 데이터를 하나의 API로 제공할지, 분리할지 결정 | 높음 |
| D1-2 | **날짜 조회 범위** | 과거 몇 일까지 조회 가능한지 (이력 보관 기간) | 중간 |
| D1-3 | **데이터 갱신 시점** | 하루전 예측 결과가 몇 시에 확정되는지 (예: 전일 18:00) | 높음 |
| D1-4 | **VPP 모델 결과 공유 방식** | O&M 디스플레이가 VPP 결과 DB를 직접 조회하는지, 별도 API를 통하는지 | 높음 |

### 3.2 2주 예측 — 발전소 화면

#### 3.2.1 기상 예측 영역 — 사용자가 보는 화면

| 영역 | 표시 내용 | 비고 |
|------|----------|------|
| 체크박스 | 풍속(기본 ON) / 기온 / 파고 토글 | 선택한 변수만 표시 |
| 박스플롯 | 14일 일별 풍속/기온/파고 분포 (min, Q1, median, Q3, max) | 풍속 구간별 색상 |
| 인라인 패널 | 박스플롯 일자 클릭 → 해당 일 24시간 상세 | 아래에 펼침 |

**인라인 패널 상세 (일자 클릭 시):**

| 영역 | 표시 내용 |
|------|----------|
| 풍속 라인 차트 | 24시간 시간별 풍속 예측 (포인트 색상: 풍속 구간별) |
| O&M 판단 테이블 | 24열(0시~23시) × 5행(종합/풍속/기온/기온저/파고) |
| 종합 상태 | 🟢 O&M 가능 / 🟡 조건부 / 🔴 불가 (시간별) |
| 개별 변수 | ✓ 충족 / ✗ 미충족 (호버 시 수치 표시) |

**O&M 판단 기준:**

| 변수 | 기준 | 미충족 시 |
|------|------|----------|
| 풍속 | ≤ 10 m/s | 🔴 O&M 불가 |
| 파고 | ≤ 1.5 m | 🔴 O&M 불가 |
| 최고기온 | ≤ 30°C | 🟡 조건부 |
| 최저기온 | ≥ 5°C | 🟡 조건부 |

**필요 API 데이터:**

| # | 항목 | 단위 | 시간 해상도 | 비고 |
|---|------|------|-----------|------|
| 1 | 일별 풍속 통계 | m/s | 14일 × {min, q1, median, q3, max} | 박스플롯용 |
| 2 | 일별 기온 통계 | °C | 14일 × {min, q1, median, q3, max} | 박스플롯용 |
| 3 | 일별 파고 통계 | m | 14일 × {min, q1, median, q3, max} | 박스플롯용 |
| 4 | 시간별 풍속 | m/s | 24시간 × 14일 | 인라인 패널 라인 차트 |
| 5 | 시간별 기온 | °C | 24시간 × 14일 | O&M 판단 테이블 |
| 6 | 시간별 파고 | m | 24시간 × 14일 | O&M 판단 테이블 |

> **📌 질문**: 박스플롯 통계(#1~#3)를 서버에서 계산하여 제공할지, 시간별 raw 데이터(#4~#6)만 제공하고 프론트에서 계산할지?  
> → 시간별 raw 데이터만 제공하면 박스플롯 + 인라인 패널 모두 커버 가능

#### 3.2.2 발전량 예측 영역 — 사용자가 보는 화면

| 영역 | 표시 내용 | 비고 |
|------|----------|------|
| KPI 카드 (좌 1/4) | 1주차 총 발전량, 2주차 총 발전량 (GWh) | 2개 카드 |
| 라인 차트 (우 3/4) | 일일 총 발전량 (14일) | 경고 배너 포함 |

**필요 API 데이터:**

| # | 항목 | 단위 | 시간 해상도 | 비고 |
|---|------|------|-----------|------|
| 7 | 일일 총 발전량 | MWh | 1일 × 14 | 라인 차트 |
| 8 | 주간 총 발전량 | GWh | 1주 × 2 | KPI (= #7 합산) |

#### 3.2.3 DDI 결정/요청 사항

| # | 항목 | 설명 | 우선순위 |
|---|------|------|---------|
| D2-1 | **박스플롯 데이터 형식** | 서버에서 5분위수(min/q1/median/q3/max) 계산 제공 vs 시간별 raw 데이터 제공 후 프론트 계산. **권장: 시간별 raw 제공** (인라인 패널에서도 사용) | 높음 |
| D2-2 | **O&M 판단 로직 위치** | 서버에서 종합 상태(🟢🟡🔴)를 계산하여 제공할지, 프론트에서 기준값 기반 판단할지. **권장: 프론트 판단** (기준값 변경 유연성) | 중간 |
| D2-3 | **기상 데이터 일괄 조회** | 14일 × 24시간 = 336시간 데이터를 한 번에 조회하는 API 필요. 페이지 로드 시 1회 호출 | 높음 |
| D2-4 | **파고 데이터 제공 방식** | 공주대 수식 `파고 = f(풍속)` 적용을 서버에서 할지, 프론트에서 할지. **권장: 서버** (수식 변경 시 프론트 배포 불필요) | 중간 |

### 5.4 API 엔드포인트 제안 (초안)

아래는 O&M 결과 디스플레이에 필요한 API 엔드포인트 초안입니다. DDI와 협의하여 확정합니다.

```
# 하루전 예측
GET /api/om/shortterm/{plant_id}/{date}
  → { plant: { hourly_power[], hourly_wind[] }, turbines: { WTG01: {...}, ... } }

# 2주 예측 — 발전소
GET /api/om/midterm/{plant_id}/plant
  → { weather: { hourly_wind[14][24], hourly_temp[14][24], hourly_wave[14][24] },
      power: { daily_total[14], weekly_total[2] } }

# 2주 예측 — 터빈별
GET /api/om/midterm/{plant_id}/turbines
  → { WTG01: { daily_power[14], daily_wind[14] }, WTG02: {...}, ... }

# 3개월 예측
GET /api/om/longterm/{plant_id}
  → { weather: { weekly_boxplot[12], weekly_distribution[12], daily_summary[84] },
      power: { weekly_total[12], monthly_total[3] } }
```

> 위 구조는 제안이며, DDI 기존 API 패턴에 맞춰 조정 가능합니다.

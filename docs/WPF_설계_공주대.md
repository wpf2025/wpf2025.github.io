# WPF 예측시스템 — 공주대 설계 문서

> 작성일: 2026-03-19  
> 담당: 공주대  
> 역할: WRF 시스템 구축, 기상 예측, 파고 예측 시스템

---

## 1. 역할 범위

| 항목 | 내용 |
|------|------|
| WRF 모델 운영 | GFS-WRF 기상 예측 결과 생산 및 제공 |
| 파고 예측 수식 | 풍속 기반 파고 추정 수식 `파고 = f(풍속)` 제공 |
| 데이터 제공 | WPF 시스템(DDI)에 기상/발전량 데이터 전달 |

---

## 2. 장기 예측 (3개월) — WRF 모델

### 2.1 제공 데이터

| 시간 단위 | 데이터 항목 | 용도 (WPF 시스템) |
|-----------|-------------|-------------------|
| 시간 | 풍속, 기온, 파고 | Boxplot 차트 원본 |
| 일간 | 평균풍속, 최고기온, 최저기온, 평균파고 | O&M 가능 여부 판단 |
| 주간 | 풍속/기온/파고 분포 | Donut 차트 |
| 주간 | 발전량 | 발전량 Line 차트 |

### 2.2 모델 운영

- WRF 모델 결과 + 후처리
- WPF 서버에서 모델 계산 X (WRF 서버에서 결과만 전달)
- 추론주기: 1회/일

### 2.3 데이터 전달 방식

| 방향 | 내용 |
|------|------|
| Out → DDI | 위치정보 기반 기상/발전량 데이터 (기존 SCADA 수집 방식과 동일) |

---

## 3. 중기 예측 (2주) — GFS-WRF 연계

### 3.1 제공 데이터

| 시간 단위 | 데이터 항목 | 용도 (WPF 시스템) |
|-----------|-------------|-------------------|
| 시간 | 풍속, 기온 | 기상 예측 앙상블 입력 |

- DDI가 OpenMeteo API(ECMWF, ICON 등)와 GFS-WRF 결과를 앙상블하여 사용

### 3.2 데이터 전달 방식

| 방향 | 내용 | 비고 |
|------|------|------|
| Out → DDI | GFS-WRF 결과값 | 신규 연결 경로 협의 필요 |

---

## 4. 파고 예측 수식

### 4.1 요구사항

- 풍속 기반 파고 추정 수식 제공: `파고 = f(풍속)`
- 중기 예측에서 OpenMeteo에 파고 데이터가 없으므로, 풍속으로부터 파고를 추정하는 데 사용

### 4.2 적용 범위

| 예측 구간 | 적용 |
|-----------|------|
| 장기 (3개월) | WRF 모델에서 파고 직접 제공 |
| 중기 (2주) | `파고 = f(풍속)` 수식으로 추정 |

---

## 5. O&M 판단에 사용되는 기상 기준 (참고)

공주대 제공 데이터가 아래 기준으로 O&M 가능 여부 판단에 활용됨:

| 변수 | 기준 |
|------|------|
| 풍속 | ≤ 10 m/s |
| 최고기온 | ≤ 30°C |
| 최저기온 | ≥ 5°C |
| 파고 | ≤ 1.5m |

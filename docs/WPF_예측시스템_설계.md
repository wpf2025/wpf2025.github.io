# WPF 풍력 발전량 예측 시스템 설계

> 작성일: 2026-03-19  
> 상태: 초안

## 1. 사이트 전체 구성

| 영역 | 목적 | 비고 |
|------|------|------|
| VPP 입찰 (초단기/단기) | 현재 WPF 개선 | 기존 시스템 |
| O&M 스케줄 수립 (단기/중기/장기) | 신규 모델 + UI 개발 | 이번 개발 범위 |

관련 조직: DDI, WPC, 공주대, 내부

---

## 2. 공통 정의

### 2.1 O&M 가능 여부 판단 기준

| 변수 | 기준 |
|------|------|
| 풍속 | ≤ 10 m/s |
| 최고기온 | ≤ 30°C |
| 최저기온 | ≥ 5°C |
| 파고 | ≤ 1.5m |

### 2.2 O&M 종합 상태 (장기/중기 공통)

| 조건 | 색상 | 의미 |
|------|------|------|
| 풍속 ✓ + 파고 ✓ + 기온 ✓ | 🟢 초록 | O&M 가능 |
| 풍속 ✓ + 파고 ✓ + 기온 ✗ | 🟡 노랑 | 조건부 가능 (기온 주의) |
| 풍속 ✗ 또는 파고 ✗ | 🔴 빨강 | O&M 불가 |

### 2.3 기상 변수 구간 및 차트 색상

**풍속 (5구간)**

| 구간 | 범위 | 색상 |
|------|------|------|
| 매우 약풍 | 0 ~ 3 m/s | SkyBlue |
| 약풍 | 3 ~ 6 m/s | Blue |
| 중간풍 | 6 ~ 10 m/s | Green |
| 강풍 | 10 ~ 15 m/s | Orange |
| 매우 강풍 | ≥ 15 m/s | Red |

**기온 (3구간)**

| 구간 | 범위 | 색상 |
|------|------|------|
| 저온 | ≤ 5°C | SkyBlue |
| 적정 | 5 ~ 30°C | Green |
| 고온 | ≥ 30°C | Red |

## 3. 장기 예측 (3개월)

### 3.1 UI

| 카테고리 | 차트 |
|----------|------|
| 발전량 | 월간 총 발전량 Bar, 주간 총 발전량 Line |
| 기상 (체크박스 토글) | 주별 풍속/기온/파고 Boxplot |
| 주차별 상세 기상 | 풍속/기온/파고 분포 Donut |
| 주간 일별 상세 | O&M 가능 여부 판단 테이블 |

### 3.2 필요 데이터

| 시간 단위 | 데이터 | 용도 |
|-----------|--------|------|
| 시간 | 풍속, 기온, 파고 | Boxplot 원본 |
| 일간 | 평균풍속, 최고기온, 최저기온, 평균파고 | O&M 판단 |
| 주간 | 풍속/기온/파고 분포 | Donut Chart |
| 주간 | 발전량 | Line Chart |

### 3.3 모델

- WRF 모델 결과 + 후처리 (WPF 서버에서 모델 계산 X)
- 추론주기: 1회/일

### 3.4 I/O

| 방향 | 내용 |
|------|------|
| In | WRF 서버에서 위치정보 기반 기상/발전량 데이터 수집 (기존 SCADA 수집 방식과 동일) |
| Out | 기상예측 + 발전량 전체 데이터 API 출력 |

## 4. 중기 예측 (2주) — 발전소

### 4.1 UI

| 카테고리 | 차트 |
|----------|------|
| 발전량 | 주간 총 발전량 Text, 일일 총 발전량 Line |
| 기상 (체크박스 토글) | 일별 풍속/기온/파고 Boxplot |
| 시간별 상세 기상 | 시간별 풍속 예측 Line |
| 24시간 상세 | O&M 가능 여부 판단 |

### 4.2 필요 데이터

| 시간 단위 | 데이터 | 용도 |
|-----------|--------|------|
| 시간 | 평균풍속, 최고기온, 최저기온, 평균파고 | O&M 판단 + Boxplot 원본 |
| 일간 | 총 발전량 | Line Chart |

### 4.3 모델

| 모델 | 설명 | 담당 |
|------|------|------|
| 기상 (데이터모델) | 기존 하루전 모델 → 2주 확대 | WPF 학습/추론 |
| 발전량 (데이터모델) | 기존 하루전 모델 → 2주 확대 | WPF 학습/추론 |
| 발전소 전체 모델 | 추가 개발 필요 | 펜타 (~4/13 논의) |
| GFS-WRF 기상 | WRF 서버 결과값 수집 | WRF 서버 |

- 기상 소스: OpenMeteo API (ECMWF, ICON 등 앙상블)
- 파고: 공주대 제공 `파고 = f(풍속)` 수식 적용
- 학습주기: 1회/월 · 추론주기: 1회/일

### 4.4 I/O

| 방향 | 내용 |
|------|------|
| In | NWP 2주치 (OpenMeteo API, 기존 하루전과 동일) |
| In | SCADA (Dive 경로정보, 기존 하루전 데이터 사용) |
| In | GFS-WRF (신규 연결) |
| Out | 기상예측 + 발전량 전체 데이터 API 출력 |

## 5. 중기 예측 (2주) — 터빈별

### 5.1 UI

| 카테고리 | 차트 |
|----------|------|
| Overview 히트맵 | 14일 × 20호기 일 발전량 상대비교, 툴팁(일 총 발전량 + 일 평균 풍속) |
| Overview 랭킹 | 14일 총 발전량 순위 가로바 + 스파크라인 |
| 상세 페이지 | 주차별 총 발전량 Text, 일간 발전량 Line, 일 평균 풍속 Line |

### 5.2 필요 데이터

| 시간 단위 | 데이터 | 용도 |
|-----------|--------|------|
| 일간 | 터빈별 평균 풍속 | 히트맵 툴팁, 상세 Line |
| 일간 | 터빈별 총 발전량 | 히트맵, 랭킹, 상세 Line |
| — | ⚠️ 평균 대비 90% 미만 플래그 | 이상 터빈 표시 |

### 5.3 모델

| 모델 | 설명 | 담당 |
|------|------|------|
| 기상 (데이터모델) × 20 | 기존 하루전 모델 → 2주 확대 | WPF 학습/추론 |
| 발전량 (데이터모델) × 20 | 기존 하루전 모델 → 2주 확대 | WPF 학습/추론 |
| 터빈별 모델 × 20 | 물리모델(후류) + 데이터모델 | 펜타 (~4/13 논의) |

- 학습주기: 1회/월 · 추론주기: 1회/일

### 5.4 I/O

| 방향 | 내용 |
|------|------|
| In | 발전소와 동일 (NWP, SCADA, GFS-WRF) |
| Out | 터빈별 기상예측 + 발전량 API 출력 |

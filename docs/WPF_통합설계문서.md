## 1. 업그레이드 개요

### 1.1 배경

기존 풍력 발전량 예측 시스템에 O&M 예측 모델이 추가되고, 관리 대상 발전소가 확대됨에 따라 시스템 업그레이드가 필요합니다.

- 기존 VPP 예측 모델에 **O&M용 예측 모델 추가**
- 관리 대상 **발전소 확대** (서남해해상풍력, 월정육상풍력, 탐라해상풍력)
- 늘어난 모델과 발전소를 **효과적으로 관리**할 수 있는 UI 필요

### 1.2 업그레이드 방향

1. 기존 메뉴 구조는 유지하고, 세부 페이지를 업데이트
2. O&M 예측 결과 메뉴 신규 추가
3. VPP 예측 결과 메뉴 신규 추가
4. 홈 대시보드 개선

## 2. 시스템 구성

### 2.1 영역별 구성

| 영역 | 목적 | 상태 |
|------|------|------|
| VPP 입찰 (초단기/단기) | 전력시장 입찰용 발전량 예측 | 기존 시스템 개선 |
| O&M 스케줄 수립 (단기/중기/장기) | 정비 계획 수립용 기상/발전량 예측 | 신규 개발 |
| 대시보드 | 통합 모니터링 및 결과 조회 | 신규 개발 |

### 2.2 기능 매트릭스

| 기능 | VPP (초단기) | VPP (단기) | O&M (단기) | O&M (중기) | O&M (장기) | 상태 |
|------|:---:|:---:|:---:|:---:|:---:|------|
| 모델 소스 등록 | ● | ● | — | ● | — | 기존 개선 (O&M 중기 모델 추가) |
| 모델 학습 | ● | ● | — | ● | — | 기존 개선 |
| 모델 추론 | ● | ● | — | ● | — | 기존 개선 |
| 결과 디스플레이 | ● | ● | ● | ● | ● | VPP 기존 / O&M 신규 |
| 입력데이터 인터페이스 | ● | ● | — | ● | ● | 기존 + 신규 |
| 출력데이터 API | ● | ● | ● | ● | ● | 기존 + 신규 |

> ● = 해당, — = 해당 없음  
> O&M 단기: VPP 하루전 모델을 그대로 사용 (O&M용 결과 디스플레이만 추가)  
> O&M 장기: CFS-WRF 서버 결과 수집만 (WPF 서버 모델 없음)

## 3. 메뉴 구조

### 3.1 기존 메뉴 (유지)

| # | 메뉴 | 서브메뉴 | 비고 |
|---|------|---------|------|
| 0 | Source 관리 | Source 관리 | 세부 페이지 업데이트 |
| 1 | Model 관리 | Data 수집 관리, Model 학습 관리 | 세부 페이지 업데이트 |
| 2 | 예측 관리 | 기상 예측 관리, 발전량 예측 관리 | 세부 페이지 업데이트 |
| 3 | Interface 관리 | Interface 관리 | 세부 페이지 업데이트 |
| 4 | 모니터링 | 실행 로그 모니터링, 연동 로그 모니터링 | 세부 페이지 업데이트 |
| 5 | 승인 관리 | 승인 관리 | 세부 페이지 업데이트 |
| 6 | 시스템 관리 | 기준 정보 관리, 공통코드 관리, 서버 환경 설정, 사용자 정보 관리 | 세부 페이지 업데이트 |

## 5. 홈 대시보드

시스템 전체 현황을 한눈에 파악할 수 있도록 4개 영역의 **대표값만 표시**합니다.  
각 영역에서 발전소를 클릭하면 해당 메뉴의 세부 정보 페이지로 이동합니다.

### 5.1 레이아웃

```
┌──────────────────────┬──────────────────────┐
│  서버 자원 현황        │  모델 학습/추론        │
│  CPU / MEMORY / DISK  │  등록 모델 수          │
│                       │  실행 TASK 수          │
│                       │  최근 학습 시간         │
├───────────────────────┴──────────────────────┤
│  🌀 O&M 예측 현황                              │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐      │
│  │ 서남해     │ │ 월정      │ │ 탐라      │      │
│  │ 풍속 9.3  │ │ 풍속 7.1  │ │ 풍속 11.2 │      │
│  │ D+1 780   │ │ D+1 28   │ │ D+1 420  │      │
│  │ 2주 4.2   │ │ 2주 0.2  │ │ 2주 2.1  │      │
│  └──────────┘ └──────────┘ └──────────┘      │
│  → 발전소 클릭 시 O&M 발전소 상세로 이동        │
├──────────────────────────────────────────────┤
│  ⚡ VPP 예측 현황                               │
│  발전소 │ MAE DA │ MAE RT │ NMAE DA │ NMAE RT │
│  월정   │ 1.82   │ 1.15   │ 5.2%    │ 3.8%    │
│  탐라   │ 2.03   │ 1.42   │ 6.1%    │ 4.2%    │
│  → 발전소 클릭 시 VPP 발전소 상세로 이동        │
└──────────────────────────────────────────────┘
```

### 5.2 영역별 표시 항목

| 영역 | 표시 항목 (대표값) |
|------|-------------------|
| 서버 자원 현황 | CPU 사용률, MEMORY 사용률, DISK 사용률 |
| 모델 학습/추론 | 등록 모델 수, 실행 TASK 수, 최근 학습 시간, 드리프트 경고 수 |
| O&M 예측 현황 | 발전소별: 평균 풍속(m/s), D+1 예측 발전량(MWh), 2주 예측 발전량(GWh) |
| VPP 예측 현황 | 발전소별: MAE DA/RT(m/s), NMAE DA/RT(%), 수집 상태, 전송 상태 |

## 6. O&M 예측 결과 (신규)

### 6.1 발전소 현황 (첫 페이지)

3개 발전소의 현재 상태를 카드 형태로 비교합니다.

| 항목 | 내용 |
|------|------|
| 발전소 카드 | 서남해해상풍력 / 월정육상풍력 / 탐라해상풍력 |
| 카드 내 표시 | 현재 출력(MW), 오늘 누적(MWh), 평균 풍속(m/s), D+1 예측, 2주 예측 |
| 하단 차트 | 일별 발전량 추이 (30일, 3개 발전소 비교), 설비 이용률 추이 |
| 동작 | 카드 클릭 → 해당 발전소 상세 페이지로 이동 |

### 6.2 발전소 상세 페이지

상단에 **발전소 전환 탭**과 **내용 탭**으로 구성됩니다.

```
┌─────────────────────────────────────────────┐
│  ← 전체 비교  │  [서남해] [●월정] [탐라]     │  ← 발전소 전환
├─────────────────────────────────────────────┤
│  [Overview] [하루전 예측] [2주 예측] [3개월 예측] │  ← 내용 탭
└─────────────────────────────────────────────┘
```

> 페이지 세부 내용은 **wpf2025.github.io** 목업 페이지를 참조합니다.

#### 6.2.1 Overview 탭

| 영역 | 내용 |
|------|------|
| 좌측 (2/3) | 오늘 시간별 예상 풍속 차트, 오늘 시간별 예상 발전량 차트 |
| 우측 (1/3) | 오늘 기상예보 (날짜, 평균풍속, 풍속/기온/파고 범위), 발전소 정보 |

#### 6.2.2 하루전 예측 탭

| 서브탭 | 내용 |
|--------|------|
| **발전소** | 시간별 발전량(bar) + 풍속(line) 복합 차트, 내일 예상 총 발전량 KPI |
| **터빈별** | WTG #1 ~ #20 개별 터빈 상세 예측 (발전량+풍속 듀얼 차트, 높이 220px) |

| 항목 | 내용 |
|------|------|
| 결과 디스플레이 상태 | 신규 (Mock-Up 참고) |
| 데이터 소스 | VPP 하루전 모델 공유 (O&M용 결과 디스플레이만 추가) |

#### 6.2.3 2주 예측 탭 — 발전소

| 카테고리 | 차트 타입 | 데이터 소스 |
|----------|-----------|-------------|
| 발전량 | 주간 총 발전량 Text KPI + 일일 총 발전량 Line | WPF 서버 모델 |

| 24시간 상세 | O&M 가능 여부 판단 테이블 | WPF 서버 모델 |

#### 6.2.4 2주 예측 탭 — 터빈별

히트맵 상세:
- 날짜별 일 발전량 상대 비교 (5단계 팔레트)
- 툴팁: 일 총 발전량 + 일 평균 풍속
- 터빈 라벨 클릭 → 상세 페이지 이동
- ⚠️ 평균 대비 90% 미만 터빈 플래그 표시

#### 6.2.5 3개월 예측 탭

## 8. 데이터 연계

### 8.1 입력데이터 인터페이스

| 데이터 소스 | 사용 영역 | 방식 | 상태 |
|-------------|-----------|------|------|
| OpenMeteo API (ECMWF, ICON 등) | VPP 단기, O&M 중기 | REST API | 기존 |
| SCADA (Dive 경로) | VPP, O&M 중기 | 경로정보 | 기존 |
| GFS-WRF (공주대) | O&M 중기 | 경로정보 | 신규 |
| CFS-WRF (공주대) | O&M 장기 | 경로정보 | 신규 |

### 8.2 장기 예측 데이터 흐름

| 방향 | 연계 대상 | 방식 |
|------|-----------|------|
| In | CFS-WRF 서버 (공주대) | 경로정보 기반 기상/발전량 데이터 수집 |
| Out | 프론트엔드 | API 출력 |

## 9. 공통 UI 규칙

### 9.1 O&M 종합 상태 표시

| 조건 | 색상 | 의미 |
|------|------|------|
| 풍속 ✓ + 파고 ✓ + 기온 ✓ | 🟢 초록 | O&M 가능 |
| 풍속 ✓ + 파고 ✓ + 기온 ✗ | 🟡 노랑 | 조건부 가능 |
| 풍속 ✗ 또는 파고 ✗ | 🔴 빨강 | O&M 불가 |

### 9.2 기상 변수 색상 코딩

**풍속 (5구간)**

| 구간 | 범위 | 색상 |
|------|------|------|
| 매우 약풍 | 0~3 m/s | Sky Blue |
| 약풍 | 3~6 m/s | Blue |
| 중간풍 | 6~10 m/s | Green |
| 강풍 | 10~15 m/s | Orange |
| 매우 강풍 | ≥15 m/s | Red |

**기온 (3구간)**

| 구간 | 범위 | 색상 |
|------|------|------|
| 저온 | ≤5°C | Sky Blue |
| 적정 | 5~30°C | Green |
| 고온 | ≥30°C | Red |

### 9.3 O&M 기상 평가 기준

| 변수 | 기준 | 비고 |
|------|------|------|
| 풍속 | ≤ 10 m/s | 초과 시 O&M 불가 |
| 최고기온 | ≤ 30°C | 초과 시 조건부 |
| 최저기온 | ≥ 5°C | 미만 시 조건부 |
| 파고 | ≤ 1.5m | 초과 시 O&M 불가 |
